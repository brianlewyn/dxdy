¿Qué tan desarrollado esta el programa que reliza derivadas?

El programa que reliza derivadas está escrito tanto para JavaScript como en Golang. Se podría decir que el programa aun sigue en modo beta, ya que aún esta suscetible a errores, pero son errores que pueden ser mejorados para más adelante.

El programa que calcula las derivadas tiene una licencia del tipo "GNU Affero General Public License v3.0", por lo tanto, cualquier persona tiene acceso y puede usar el código, siempre y cuando respete lo dictado en la licencia, en el cual explica que si haces uso de el, debes aportar tu desarrollado para mejorar el software. Por el momento, solo esta accesible el código de Golang a través del paquete github.com/BrianLewyn/dy.

El paquete BrianLewyn/dy está diseñado para programadores que tienen que trabajar con derivadas matemáticas.

Es por eso que este paquete brinda funciones que nos permiten calcular derivadas de una o más expresiones matemáticas de una manera más fácil y organizada.

El objetivo es que a partir de una cadena de texto que contiene expresiones matemáticas con o sin literales, las funciones del paquete puedan devolver el resultado con el mismo tipo de datos.

Tanto en JS como GO, tienen la funciones con la características siguientes:

  • AddKxn()
    • Suma todas las expresiones kx^n siempre que tenga el mismo literal x y exponente n.
    • No hay límite en el número de expresiones a calcular.
    • Se pueden ingresar valores enteros o flotantes tanto para la constante k como para el exponente n.
    • Si usa flotantes, el valor tendrá una precisión de dos decimales, pero esto se puede cambiar.
    • Puedes trabajar con cualquier literal con tantas letras como quieras.
    • Ordena las expresiones kx^n de mayor a menor exponente.
    • Los signos positivo y negativo se tienen en cuenta al sumar.
    • Entre cada expresión matemática puedes poner un espacio x+x o un espacio, un signo y otro espacio x+x.
    • Si al comienzo de la cadena, una expresión es negativa, el signo menos se coloca como está. Pero si es positivo, se puede poner el signo, el signo y un espacio, o simplemente nada.
    • No importa cómo ingrese kx^n, siempre que sea una expresión, el programa se encargará de inferir y reconstruir las expresiones a su forma base para realizar la operación correcta.
    • Todas las expresiones matemáticas que tienen constante 1, exponente 1 o 0, se representarán en su forma más compacta y clara.

  • DyKxn()
    • Mantiene todas las características anteriores.
    • Para cada expresión ejecuta la regla de la derivada de kx^n.
    • En el posible caso que defina el literal en mayúsculas y todas las expresiones en minúsculas o viceversa, el programa siempre pondrá todos los literales en minúsculas.
    • Nota:Si todas las expresiones se escribieron exactamente con la misma literal y no hay otro tipo de error todo saldrá bien, pero de lo contrario, se arroja un error como si hubiera un literal constante.

Note: A lo largo de todo el programa, se encuentran notas muy breves, pero que aportan mucho valor para saber que hace cada parte del código, todo esto escrito en inglés por si aun extranjero le interesa saber como funciona el programa.
